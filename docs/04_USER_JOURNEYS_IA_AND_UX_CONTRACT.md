# 04｜用户旅程、信息架构与体验合同

状态：`APPROVED_FOR_BUILD`  
版本：V0.1  
日期：2026-07-20  
文档 Owner：Design Owner + Product Owner  
前置决定：`DEC-004`、`DEC-006`、`DEC-011` 已批准

## 1. 体验北极星

用户在任何时刻都能回答四个问题：

1. 我是谁，以什么身份进入？
2. 我现在处于哪一步？
3. 当前唯一有效行动是什么？
4. 完成后由谁反馈、下一步是什么？

界面不向用户暴露系统版本、兼容迁移、领域事件、内部状态拼接或旧系统概念。

## 2. P0 核心旅程

### JRN-001｜标准通过路径

```text
收到邀请
  → 打开加入页并确认用途
  → 完成最小身份确认
  → 进入“当前行动”
  → 阅读任务与完成标准
  → 上传/填写并提交
  → 看到“已提交，等待谁反馈”
  → 主管在授权队列开始评审
  → 主管提交 PASS
  → 新人看到结果、核心反馈和唯一下一步
  → 完成交接/结束探索营
```

### JRN-002｜需要修订路径

```text
提交
  → 主管提交 REVISION_REQUIRED
  → 新人看到当前反馈与“修订”主行动
  → 旧版本只读，新草稿基于当前版本
  → 再提交形成新版本
  → 新评审固定引用新版本
  → PASS 或再次修订
```

### JRN-003｜身份异常路径

```text
邀请无效/过期/撤销
  → 说明原因
  → 不创建半成品 Enrollment
  → 提供联系运营或重新邀请的唯一恢复动作
```

### JRN-004｜提交冲突与网络失败

```text
提交超时/响应丢失
  → 保留草稿
  → 使用原 idempotency key 查询/重试
  → 已成功则返回原结果；未成功才执行一次
  → 冲突时显示最新事实并要求用户重新确认
```

### JRN-005｜评审材料不完整

```text
主管打开评审
  → 服务端报告缺失项
  → 可以查看现有内容，不能 finalize
  → 运营/学员收到明确修复动作
  → 完整后继续同一授权评审
```

### JRN-006｜通知失败

```text
业务状态已成功迁移
  → 通知异步失败
  → 用户刷新仍看到正确业务结果
  → 运营看到通知重试/失败，不修改业务结论
```

## 3. 信息架构

P0 只允许以下 canonical 页面。名称是建议值，原型评审后可调整；禁止添加 legacy/alias 页面。

| 路由 | 角色 | 核心问题 | 主行动 |
| --- | --- | --- | --- |
| `/` | 公开 | 这是什么，我如何进入？ | 使用邀请进入/登录 |
| `/join` | 受邀新人 | 这个邀请是否有效，我要确认什么？ | 确认并继续 |
| `/app` | Learner | 我现在该做什么？ | 按状态继续当前行动 |
| `/app/tasks/{assignmentId}` | Learner | 这项任务要交付什么？ | 开始/保存/提交中的唯一当前动作 |
| `/app/result` | Learner | 当前结果意味着什么，下一步是什么？ | 修订或继续交接 |
| `/review` | Reviewer | 现在先评谁？ | 开始最高优先级评审 |
| `/review/{reviewId}` | Reviewer | 这份固定版本如何反馈？ | 提交最终结论 |
| `/ops/invites` | Operator | 如何邀请/撤销当前对象？ | 创建邀请或处理当前异常 |
| `/ops/enrollments` | Operator | 哪个 Enrollment 需要处理？ | 执行明确运营命令 |
| `/ops/config` | Admin/Operator | 当前已发布任务与规则是什么？ | 发布一个版本化配置 |

约束：

- 邀请 token 首次交换后立即从地址栏移除；日志与分析不得记录完整 token。
- 角色无权访问页面时返回明确拒绝或安全入口，不渲染后再隐藏。
- `/app` 是学员唯一首页；不建立 dashboard、map、today、learning 等平行首页。
- `/review` 是主管唯一工作台；不再建立 admin review、NPC review 等同义入口。

## 4. 全局导航

### Learner

P0 不使用复杂全局导航。顶栏只包含产品身份、本人、帮助/退出；当前行动和历史入口由页面状态决定。历史只作为次级只读入口，不与当前行动并列。

### Reviewer

仅包含待办队列、已处理记录、帮助/退出。默认落在待办队列。

### Operator

包含邀请、Enrollment、配置、异常/审计。Admin 能力不混入 Learner 或 Reviewer 页面。

## 5. 页面模板

### UX-TPL-01｜当前行动页

首屏最多包含：

1. 当前阶段/状态；
2. 当前目标；
3. 反馈责任人与时间预期；
4. 唯一主 CTA。

规则、历史、完整阶段地图和证据血缘位于次级披露区域。

### UX-TPL-02｜任务与提交页

顺序固定为：任务目标 → 完成标准 → 必需材料 → 工作区/附件 → 校验与主动作。Rubric 可见但不抢占工作区。

### UX-TPL-03｜评审工作台

队列显示身份、等待时长、优先原因和材料完整性。详情固定显示本次提交版本、任务标准、证据、AI 建议（如有）、人工评价区和唯一 finalize 动作。

### UX-TPL-04｜结果与修订页

先显示权威结论和核心反馈，再显示当前主行动；旧版本、完整评分和事件时间线进入只读历史。

### UX-TPL-05｜运营列表/详情

列表只帮助定位异常或当前对象；操作必须是有意图的业务命令，禁止直接编辑数据库字段。

## 6. 状态呈现合同

| 服务端状态 | 用户文案意图 | 主行动 | 禁止 |
| --- | --- | --- | --- |
| `PENDING_IDENTITY` | 先确认身份才能开始 | 确认身份 | 同时展示任务提交 |
| `AVAILABLE` | 当前任务可以开始 | 开始任务 | 展示“已开始” |
| `IN_PROGRESS` | 继续完成当前交付 | 继续/提交 | 并列多个去向 |
| `SUBMITTED` | 已提交，等待指定反馈 | 无写动作；可查看提交 | 回到第一关、重复提交 |
| `IN_REVIEW` | 主管正在处理 | 无写动作 | 用“系统处理中”掩盖责任人 |
| `NEEDS_REVISION` | 按反馈修订当前版本 | 开始/继续修订 | 覆盖旧提交 |
| `COMPLETED` | 本阶段已通过 | 前往交接/完成 | 继续显示提交动作 |
| `BLOCKED`（CurrentAction/界面状态，不是 Assignment 枚举） | 明确说明阻塞原因与责任人 | 唯一恢复/联系动作 | 空白页或泛化报错 |

前端不得从多个字段组合出额外业务状态。若页面需要的状态不在合同中，先修改领域状态机，不得添加本地布尔值补丁。

## 7. 交互合同

- 可行动状态恰好一个主 CTA；次要按钮不得改变核心状态。
- 所有写动作都有 pending、成功、可恢复失败和冲突反馈。
- 防重复点击不能只依赖按钮 disabled；服务端必须幂等。
- 离开含未提交草稿的页面前提醒；冲突时不清空用户输入。
- GET 请求无业务副作用；打开详情不能自动开始评审或改变状态。
- 错误文案包含用户语言和 request ID，不显示堆栈/内部枚举。
- destructive 操作（撤销邀请、取消 Enrollment）二次确认并要求原因。
- 焦点在打开错误/对话框/进入工作区后移动到合理位置。

## 8. 响应式与可访问性

- 目标视口：390、768、1280；主行动无需横向滚动且在合理首屏范围。
- 完整 Learner 提交和 Reviewer 评审可纯键盘完成。
- 表单错误与字段关联；焦点回到首个错误。
- 状态不只依靠颜色；图标配文字或可访问名称。
- 对话框、菜单、披露组件支持 ESC、Tab 顺序和焦点恢复。
- 文件上传支持选择、删除、重传、进度和失败恢复。
- 文本缩放至 200% 不丢失核心功能。

## 9. 原型交付物

G0 需完成低保真状态原型，不追求完整视觉：

- [ ] Learner 首次进入、当前任务、已提交、需要修订、通过；
- [ ] Reviewer 队列、详情、材料缺失、冲突、提交成功；
- [ ] Operator 邀请、撤销、主管分配、异常处理；
- [ ] 加载、空、无权限、404/410、系统故障、离线/超时；
- [ ] 390/768/1280 与键盘路径；
- [ ] 5 秒测试和首次任务测试脚本。

## 10. 体验验收

- `AT-UX-001`：目标角色 5 秒内正确说明当前行动；
- `AT-UX-002`：Learner 两次交互内开始有效任务；
- `AT-UX-003`：标准路径与修订路径无重复状态/主动作；
- `AT-UX-004`：Reviewer 首屏可识别最高优先级并开始；
- `AT-UX-005`：材料不完整时无法误提交结论；
- `AT-UX-006`：三视口无遮挡、溢出和动作丢失；
- `AT-UX-007`：主路径纯键盘完成；
- `AT-UX-008`：网络超时、409 冲突和通知失败不丢失事实/草稿；
- `AT-UX-009`：所有旧路径不进入 vNext 业务页面。
